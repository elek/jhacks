Oracle

http://otn.oracle.com/

"Az adatbazis motorok luxuslimuzinja"

Az Oracle Corp hosszu multra visszatekinto RDBMSe, amely vezeti az RDBMS piacot feature-k teruleten. Szinte minden komoly feature megvan benne, raw device support, stored views, java es PL/SQL proclang, klassz indexelesi lehetosegek, XML, Spatial, Text es egyeb extenderek, satobbi.

Hatranya:

  • Tul sok sajatossag es elteres a szokasoktol/szabvanytol. Nagyon konnyen keveredhetsz vendor lock-in-be.
  • Az uj feature-ok hasznalhatatlanok. De legalabbis erosen bugosak. Az Oracle politikaja az, hogy kiadnak valamit felkeszen, aztan ha van ra igeny, akkor majd kidolgozzak. Ergo: eles kornyezetben uj feature hasznalata tilos.

Ami szamunkra erdekes: Oracle JDeveloper. Nem igazan user friendly, de a szalitott osztalykonyvtarak szamat tekintve az egyik legerosebb IDE. Gyakorlatilag minden olyan eles featuret ad, amit ma hasznalni erdemes, raadasul ugy, hogy uzleti kornyezetben is bevetheto dolgokat kapsz.

Persze itt is igaz, hogy bughalom az egesz, meg a szokasos Oracle vendor lock-in, de erdemes megismerni. Egyebkent free.
