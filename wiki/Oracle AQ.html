Oracle Advanced Queueing

Egy MOM az Oracle RDBMS termekbe epitve, gyakorlatilag egy tucat PL/SQL eljarasbol megepitve. Egy sajatos, JMS-t is implementalo java APIn keresztul lehet elerni, persze csak a standard feature-ket, a tobbit Oracle specifikus APIn keresztul.

nehany erdekesseg/tapasztalat:

  • Az AQ sajat jms extensionjaval meg tudod azt oldani hogy az egy tranzakcioban bekommitolt uzeneteket egy csoportkent szeded ki. Jol hangzik mi? :) Persze ehhez valami falabu API-t irtak a oracle sracok, de mukodik.
  • Nyilvan PL/SQL bol is tudsz messageket betolteni, igy pl egy idiota autokommitos tranzakcios webappot is tulelhet az appod. Mar ha tul lehet elni ekkora marhasagot :)
  • Nem egy JMS szerver, hanem egy standalone, Oracle specifikus dolgokkal peldanyosithato JMS api ami az adatbazishoz fer hozza (igen, peldanyositashoz kell egy JDBC kapcsolat). Ennek kovetkezteben az alkalmazas szerver integraciot csinald magad modon :( weblogichoz van egy adapter de inkab ne is lenne, szoval mondhatni hogy oc4jn kivul semmi nem mukodik egyut vele, leszamitva olyan dolgokat mint mule. (dehat a mule mindennel)

Amugy mokas kis cucc, meg jo lassu :) Az Oracle 9i ota minden adatbazisban benne van. Egy ilyet igazabol nem lenne nagy kunszt megirni PostgreSQLre sem.
