java 1.7/native xml support

Egy uj feature a java 1.7ben, ami a java 1.5 annotations ujitasat hasznalja arra hogy egy mappinget nyujtson XML es beanek kozott. Vajon minek ilyet egy nyelv magjaba epiteni?

Regi elv az hogy az adatreprezentaciot kulonitsuk el a programlogikatol, es nem egy rossz otlet, ez a megoldas amolyan soft link a ketto kozott. A Bean ismeri a sajat reprezentaciojat.

Mire nem kellene ezt a dolgot hasznalni:

  • Az XML adatbazisok egy nagy adag szivas, az XQL is tobbnyire alulrol kozeliti az SQL lehetosegeit. Szoval adatbazisokhoz, nagyobb meretu adattarolashoz NEEEE!!!

Mire lehetne talan hasznalni:

  • Konfiguracios objektumok? (csak kerdezem, meg nem talaltam ki mire lesz jo)
