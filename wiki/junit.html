junit

(http://junit.org/)

Szoval a junit alapveto eszkoz egy onmagara valamit is ado programozo kezeben. Bar eleg egyszerucske. A junit teszteket hajt vegre. Valahogy igy:


class TestWhatever extends TestCase {
    //Ez hivodik meg a tesztek elott
    public void setUp() {}
    //Es a tesztek utan
    public void tearDown() {}
    //Es a tesztjeid csupa public es test-tel kezdodo metodusok, pl
    publi void testAkarmi() {
        ...
    }
}

A kicsit ujabb es modernebb junit 4 a java 1.5 annotations feature-re epit, ezzel kb igy nez ki egy teszt:

class TestWhatever { //nem am extends, pojo baszkikaim!!!
    @Test // ez a teszt maga
    public void something() {
    ...
    }
    @Before
    public void elottehivdMarMegLecci() {
    }
}

Szoval a jo oreg junit utannaeredt a fiatal tronfosztoknak mint a testng. Ha lassan is. A teljesen uj pojo teszteknek szerintem az lesz a vege hogy paran beleirjak a teszteket az objektumokba magaba, tovabblepve pedig majd az eles metodusok teszt metodusokat is fognak hivni. En barmilyen allatsagot el tudok kepzelni :)

Az IDE-k altalaban tartalmaznak valamilyen supportot a junit futtatashoz, az eclipse-ben ez nagyon rulla, a tobbit nem tudom, de itt egy link hozza: http://www.junit.org/news/ide/index.htm (NetBeans pl. már simán kezeli)

A junit mindenféle portjai:

  • dbunit
  • httpunit
  • satobbiunit

Nekem a kedvencem az amikor egyszerűen csinálsz egy spring context-et és abból szedsz ki bean-eket amiken meghívhatsz a tesztekben metódusokat. Nagyon könnyen ujrafelhasználhatóvá teszi a teszteket.

A 4.1-es verzio ujitasa kozott a leglenyegesebb talan a Test/Categorization, szinten java annotations segitsegevel. Sok uj es modern dolog van a 4.1ben csak sajnos semmi sem tamogatja per pillanat. Se eclipse, se maven. Tul nagy szakitas volt a regi apival talan, de remelem ez lassan tovabb fejlodik.
